## Overview

![screenshot](./assets/desktop-preview.png)

A responsive, high-conversion pricing table built to execute precise UI/UX across all market viewports. Submitted as a solution for a DevChallenges project.

### What I learned

This project was a masterclass in frontend asset management and CSS balance sheets. Major structural takeaways include:

- Liquid Grid Architecture: Moving from fixed pixels to fluid fractional shares using grid-template-columns: minmax(0, 1fr) to scale assets dynamically without breaking the viewport.

- SVG Data Liquidation: Converting locked .svg image files into raw inline code using fill="currentColor" so icons automatically inherit CSS state changes.

- Advanced Parent Selectors: Deploying the modern :has() pseudo-class to trigger parent-level animations (scaling the whole card) based entirely on child-level interactions (hovering the CTA button), bypassing the need for JavaScript.

- Market Adaptation (Mobile UX): Engineering a horizontal swipe carousel with a frozen position: sticky feature column to handle data-heavy tables on restricted mobile screens.

- Fully responsive UI scaling from large desktop displays down to narrow mobile screens.

- Mobile-optimized horizontal scroll track to maintain data readability without forced layout breaks.
